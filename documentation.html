Introduction

ComponentMelt is a library designed to allow modularization of react components. It allows you to build React components that own their logic but remain structurally invisible to the application hosting them so they can be reused in any react application.

Installation

Install the library via your preferred package manager:

npm install component-melt
# or
npm install --save-dev component-melt

Developer Workflow

ComponentMelt provides a CLI to scaffold module creation. These modules shall be published to a private or public registry (your choice) and imported by any host application that satisfies their structural contract.

1. Create Your Module

Initialize a new module and add a component using the melt-gen CLI tool.

# Initialize the module
melt-gen init module-example --scope component-melt
        
# Add your functional component
cd module-example
melt-gen add TestComponent
--scope: Scope is not really needed in order to generate the modules. You only want to use it if the module needs an org prefix.

2. Define the Contract

Inside your module, declare exactly what the component needs to operate. In this example, our component requires a text-update capability for the "example" reducer.

const contract = {
    name: "TextTester",
    requiredCapabilities: {
        example: ['UPDATE_TEXT'] 
    }
}
        
export const TextTester = createComponent({
    component: TestComponent,
    contract
})

The module with the component can now be published to your registry and shared across projects.

3. Host Implementation

After installing your module use Blueprints to satisfy the contract.

In your host app, declare the capabilities before rendering. If the actionArray does not satisfy the contract, the component will not render. Instead, if no fallback is provided when using createComponent, ComponentMelt will show it's own component showing the contract violations.

import { blueprints } from 'component-melt';
import { TextTester } from '@component-melt/module-example';
        
// Host declares it can handle the module's needs
blueprints.declareCapability('example', ['UPDATE_TEXT']);
        
function App() {
    return <TextTester />;
}
